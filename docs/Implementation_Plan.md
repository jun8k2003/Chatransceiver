# チャットメッセージ（バブル）へのダイレクトリンク機能の実装計画

URLパラメータを利用して特定のメッセージ（音声・テキスト）を直接指定し、アプリ起動時にそのメッセージの所属するチャットルームを自動で開き、対象のバブルにフォーカス（スクロール・ハイライト）して自動再生する機能を実装します。

## 課題・要件の整理
*   **URL仕様**: `/?c=コミュニティID&m=メッセージID` のように指定できること。
*   **整合性チェック**: 指定されたメッセージが存在し、対象のコミュニティに所属しているか、およびユーザーがアクセス権を持っているかを確認すること。
*   **UI制御**: 対象メッセージがグループチャットか個別チャットかを自動判別してルームを選択し、画面を描画後に該当バブルへスクロールすること。
*   **自動再生の壁 (ブラウザポリシー)**: 最新のブラウザでは「ユーザーの画面操作なしの自動音声再生（Autoplay）」がブロックされる仕様があります。外部アプリ（Slackなど）からリンクを踏んで開いた直後の場合、再生がブロックされる可能性があります。ブロックされた場合は、メッセージにフォーカスを当てるのみにとどめる（または再生ボタンをハイライトする）フォールバック処理を組み込みます。

## Proposed Changes

### 1. `src/services/supabase.ts`
指定されたメッセージの情報を取得する関数を追加します。
#### [MODIFY] `supabase.ts`
*   `getMessageInfo(messageId: string)` メソッドを追加。
    *   `messages` テーブルと `chat_rooms`, `communities` を JOIN して取得。
    *   戻り値として `{ roomId, roomType, communitySlug, audioUrl }` 等を返却します。

### 2. `src/app.ts` (Appクラス)
URLパラメータの解析と初期化ロジックに組み込みます。
#### [MODIFY] `app.ts`
*   **`init()` 処理の拡張**:
    *   URLのクエリパラメータ（またはハッシュ）から `m` (メッセージID) を抽出。
    *   `m` が存在する場合、`getMessageInfo(m)` で情報を取得し、`c` パラメータ（コミュニティ）と整合性が取れているか検証。
*   **ルーム自動選択と再生予約**:
    *   コミュニティデータ (`loadCommunityData`) のロード完了後、対象の `roomId` をもとにグループチャットか個別チャットかを判別し、`handleGroupSelect` または `handleUserCheckChange` を呼び出してチャット画面を開く。
    *   その直後に、対象のメッセージIDと音声URLを引数にして、UIのスクロール命令と音声の再生処理をトリガーします。
    *   再生時、`NotAllowedError` (Autoplayブロック) が発生した場合は警告をコンソールに出力しつつ、UIのフォーカスだけは維持する仕組みにします。

### 3. `src/ui/chat.ts` (ChatWindowUIクラス)
特定のメッセージバブルを強調表示・スクロールする機能と、URL取得機能を追加します。
#### [MODIFY] `chat.ts`
*   **`render()` メソッドの拡張 (ダイレクトジャンプ用)**:
    *   引数に `targetMessageId?: string` を追加。
    *   描画完了後、`targetMessageId` に一致するDOM要素（`.chat-message[data-msg-id="..."]`）を探し、`element.scrollIntoView({ behavior: 'smooth', block: 'center' })` で画面中央へスクロールさせます。
    *   CSSクラス `.highlight-pulse` （例）を付与して数秒間ハイライトさせ、ユーザーの視線を誘導します。
*   **メッセージの右クリック (Context Menu) 対応**:
    *   メッセージ要素に `contextmenu` イベントリスナーを追加。
    *   発火時にデフォルトの右クリックメニューをキャンセル（`e.preventDefault()`）し、現在のURLに `?m=メッセージID` を付与したURLを生成。
    *   `navigator.clipboard.writeText()` を用いてクリップボードにコピーし、「リンクをコピーしました」とトーストまたはアラートで通知します。

### 4. `src/style.css`
ハイライト用のアニメーションを追加します。
#### [MODIFY] `style.css`
*   `@keyframes highlightPulse` を定義し、`.chat-message.highlight-pulse` クラスで対象バブルの背景色を一瞬点滅させるスタイルを追加します。

## User Review Required

> [!WARNING]
> **ブラウザの自動再生ブロックについて**
> 外部アプリ（Slack、LINEなど）からリンクをクリックして新しいタブで開いた場合、ブラウザのセキュリティ仕様により「ユーザーが画面を1回もクリックしていない状態」での音声再生はブロックされます。
> この場合、アプリ側としては「該当メッセージまでスクロールし、ハイライト表示する」ところまでは確実に行えますが、再生自体は失敗するため手動で再生ボタン（▶）を押してもらう必要があります。この挙動の限界についてご了承いただけますでしょうか？（回避策としては、起動直後に「再生を開始する」といったワンクッションのボタンを表示する設計などが考えられますが、まずはシンプルに「可能な限り自動再生し、ダメならハイライトのみ」とするのが自然かと思います）

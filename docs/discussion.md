## 🚧 議論中: ボット機能の追加 (2026-07-02〜)

* **仕様ドラフト**: [bot_function_spec.md](./bot_function_spec.md)（ブランチ `2026-07-02-bot-function`）
* **概要**: 「外部連携（全ユーザー向け: Discord / メール / 将来 LINE・Teams）」と「ボット振る舞い（`is_bot`: 自動返信＋ループ遮断＋🤖バッジ）」の2つの直交する概念として設計する。ボット＝通常ユーザーアカウントで、`user_inboxes` ファンアウトを起点に既存 Edge Function（`send-push-notification`）を拡張して実行する。
* **合意済みの方向性**:
  * ボット専用の参加フロー・管理画面・管理者ロールは作らない
  * 外部連携はボット専用でなく全ユーザーの機能として一元化（`user_integrations` テーブル）。既存 `discord_webhook_url` は将来そこへ移行
  * `is_bot` は「アカウントの振る舞いの規定」（外部連携とは独立）
  * ループ防止は「ボットはボットの発言に反応しない」方式（ファンアウト自体は止めない）。複数ボット同居時も安全なことをトレースで確認済み
  * Edge Function は新設せず `send-push-notification` を1本のまま拡張（自動返信が実行結果を正確に報告できる）
  * メール送信は **汎用 Webhook 連携＋GAS 自作** に決定（O-1解消、2026-07-03 改訂）。Discord Webhook と同型の「URL登録＋POST」で外部メールベンダー依存を排除。無料Gmailの上限100宛先/日で十分と判断。Resend（ドメイン必須）・Brevo（一旦決定→改訂、非開発者向け開放時の再検討候補として仕様書 §4.3 に保持）
* **次に議論すべきこと**: bot_function_spec.md §6 の Open Issues（特に O-2 自動返信のTTS冗長性、O-3 レート制御、O-4 失敗時挙動）
